# 6 混合模拟案例研究

## 6.1 引言

本章介绍了混合模拟两种新应用的案例研究。主要目的是展示和验证重新设计和新实现的对象导向混合模拟框架的功能和灵活性。这两个案例研究在加州大学伯克利分校的 $\mu \mathrm { N E E S }$ 实验室进行，并利用了混合模拟的广泛方法和实现细节。

## 6.2 快速地理分布测试

网络技术的最新进步与实验软件框架OpenFresco的结合，使快速地理分布混合模拟终于成为可能。作为George E. Brown, Jr. 地震工程模拟网络（NEES）计划的一部分，一个名为NEESgrid的国家合作实验室已经开发出来。NEESgrid基于Internet2的Abilene网络，将15个NEES设备站点连接在一起。Abilene网络是一个用于教育和研究的私有高速骨干网络，截至2006年，其带宽高达10 Gbit/sec。对于在加利福尼亚州Davis的NEESinc和加利福尼亚州Richmond的nees@berkeley设备站点的 $\mu \mathrm { N E E S }$ 实验室之间进行的快速地理分布混合模拟，使用了NEESgrid网络的1 Gbit/sec连接之一。随后描述的测试展示了OpenFresco（见第3章）如何与高性能有限元软件结合使用，以跨越大网络距离执行软实时混合模拟。

### 6.2.1 动机

在过去十年中，为改善结构在地震、爆炸和风等极端荷载条件下的行为而引入高性能装置显著增加。这些装置中的大多数，如基础隔震支座、耗能装置和高性能材料，表现出固有的速率依赖行为。因此，对包含此类装置的结构系统进行测试时以正确的加载速率进行是至关重要的。这意味着全尺度混合模拟需要实时执行。此外，如果混合模型恰好是原型模型的缩放版本，长度比例因子为 $\mathrm { S } _ { \mathrm { l } }$，并且还需要保持速度，即速度比例因子 $\mathrm { S } _ { \mathrm { v } } = 1$，则混合模拟需要以比实时快 ${ \mathrm { S } _ { \mathrm { l } } } ^ { - 1 }$ 倍的速度执行。另一方面，如果要为缩放模型结构保持应变率而不是速度，则实验测试需要再次实时执行，因为时间比例因子 $\mathrm { S } _ { \mathrm { t } } = 1$ 与长度比例因子Sl无关。

### 6.2.4 网络性能评估

为了建立分布式混合模拟的客户端和服务器程序之间的连接，OpenFresco的ShadowExpSite和ActorExpSite类使用通道对象。通道对象是一个抽象基类，定义了数据离开或进入程序的通信端点。因此，是派生子类实现了可用于在客户端和服务器程序之间传输数据的不同通信协议（套接字）。所有可用选项的客户端-服务器架构在第3章中详细描述。对于这里描述的快速地理分布混合模拟案例研究，使用TCP_Socket对象设置从客户端应用程序到服务器应用程序的持久TCP/IP连接。TCP/IP连接保证了传输的消息永远不会丢失、损坏或乱序接收。此外，持久连接提供了显著的性能提升，因为客户端-服务器握手引起的TCP连接设置延迟仅在分布式测试的初始化期间发生一次，而不是在每次单独事务开始时发生。

除了使用持久TCP/IP连接外，发送到TCP堆栈的数据包大小也显著影响了网络性能。由于TCP协议具有数据流接口，允许应用程序将任何大小的数据发送到TCP堆栈（甚至一次一个字节），因此确定产生最佳网络性能的TCP段大小势在必行。确定这种最佳段大小时存在权衡。如果TCP段太大，它们可能在IP级别被分段。另一方面，如果它们太小，网络性能会大大降低，因为少量数据与至少40字节的TCP和IP头开销一起发送。因此，通过将ShadowExpSite和ActorExpSite之间传输的数据包大小增加到超过响应向量所需的最小大小，避免了所谓的发送者愚蠢窗口综合症。发送大量微小数据包非常低效，甚至可能扰乱其他网络流量（Gourley等人，2002年）。

为了研究数据包大小对网络性能的影响，执行了一系列混合模拟，其中单跨框架模型的实验柱被模拟而不是在实验室中测试，跨越不同的网络配置。分布式混合模拟测试在nees@berkeley设备站点的局域网上以及Davis和Berkeley之间的Abilene网络上进行。对于每次测试，记录执行1599步长直接积分分析所需的总模拟时间，并将其作为评估所研究网络性能的度量。

![](images/6_6.jpg)  
图 6.6 网络性能取决于数据包大小。

从图6.6可以看出，一旦达到大约200个64位双精度的数据大小，模拟时间突然下降了约10倍。正如预期的那样，对于非常大的数据包，性能开始再次恶化，但速度比遇到之前的突然增加要慢得多。作为这些性能测试的结果，为单跨框架模型的快速地理分布混合模拟选择了256个64位双精度的向量大小。重要的是要认识到，在实时xPC Target机器上运行的事件驱动预测-校正模型（见第5章）实际上确定了在正常操作条件下混合模拟运行的速度。只要没有过度的网络和积分延迟，预测-校正模型中设置的模拟时间步就确定了每个积分时间步在挂钟上经过多少时间。

由于本案例研究的目标是实时执行混合模拟，模拟时间步 $\Delta \mathrm { t } _ { \mathrm { s i m } }$ 必须设置为等于积分时间步 $\Delta \mathrm { t } _ { \mathrm { i n t } } = 2 0$ 毫秒。然而，由于模拟时间步必须是控制器时间步 $\Delta \mathrm { t } _ { \mathrm { c o n } }$ $= ~ 1 / 1 0 2 4$ 秒的倍数，这并不完全可能。决定为每个模拟时间步选择20个子步。只要网络和积分任务能够在预定义模拟时间步的 $60 \%$ 内执行，这就产生了 Δtsim = 19.53125 毫秒。因此，理论测试速率为0.9766，结果略快于实时。

模拟时间步内不同任务的时序和交互可以从图6.7看出。事件流如下。一旦在ti处达到目标位移，就获取抗力并通过Abilene网络发送到积分任务。与此同时，信号生成任务执行响应预测，每 $\Delta \mathrm { t } _ { \mathrm { c o n } }$ 创建新的位移命令。这保证了执行器保持移动直到接收到新的目标位移。完成运动方程的直接积分后，新的位移目标从客户端通过Abilene网络发送回服务器，这触发有限状态机从预测切换到校正。如前所述，如果组合网络和积分时间超过预定义模拟时间步的 $60 \%$，预测-校正模型自动减慢执行器运动。为了实现最佳性能，这种情况需要尽可能少地发生。一旦在 $\mathrm { t } _ { \mathrm { i } + 1 }$ 处达到目标位移，就测量抗力，事件序列重新开始。

图 6.7 时间步内任务的时序和交互。

**有限元模型结果：**

图6.9显示了单跨框架结构两个自由自由度的横向位移、速度和加速度响应。响应历史从实时测试期间记录的OpenSees输出文件中绘制。如以下三个图上标记的，左实验柱顶部看到的最大位移、速度和加速度值分别为2.31 in.、22.1 in./sec和 $2 4 2 { \mathrm { ~ i n . / s e c } } ^ { 2 }$。由于左实验柱顶部连接到 $\mu \mathrm { N E E S }$ 装置的执行器，并且测试是实时执行的，执行器经历了相同的响应最大值。因此，这些极端值必须保持在执行器位移限制 $\pm 7 . 5$ in.和执行器速度限制 $\pm 2 3 . 3$ in./sec（如表6.2所列）之内，以免使响应饱和或触发联锁装置并关闭控制系统。

## 6.3 结构倒塌模拟

### 6.3.1 动机

地震工程领域的一个重要研究方向涉及模拟结构倒塌的能力，以提高我们对结构在非常罕见的地震事件期间性能的理解。然而，倒塌的实验研究通常复杂、昂贵且潜在危险。可以技术上采用前面描述的三种实验方法中的任何一种来执行结构倒塌测试。然而，准静态和振动台测试在进行、执行和解释此类测试方面都面临困难，而混合模拟则不然。

接下来总结了三种实验方法相对于结构倒塌研究的优势和劣势。

1. 虽然准静态测试相对容易且经济地进行，但施加在测试试样上的需求与观察到的损伤没有直接关系。这引发了关于试样是否测试不足或过度测试的问题，反过来使得对结构在倒塌附近或倒塌时的行为得出准确结论变得极其困难。

2. 振动台测试提供了关于特定地震动动态响应的重要数据，考虑了结构的惯性和能量耗散特性以及几何非线性、局部屈服和损伤对响应的后果。然而，对于此类测试，需要完整的结构系统，并且需要构建包括复杂主动或被动重力荷载设置的试样。此外，需要采取预防措施来保护昂贵的测试设备免受倒塌期间试样冲击的影响。这使得振动台上的倒塌研究非常复杂、昂贵且潜在危险。此外，由于模拟器平台和/或控制系统限制，通常需要缩尺或高度简化的试样，这质疑了许多结构倒塌振动台测试的真实性。   
3. 混合模拟使得通过将其纳入混合模型的解析部分来研究几何非线性、三维效应、多支座激励和土-结构相互作用成为可能。因此，混合模拟方法使研究人员能够测试大尺度或全尺度试样，并减少或避免缩放问题。此外，只有结构的关键的、对倒塌敏感的元素需要物理测试，允许在相同预算下大幅增加不同倒塌测试的数量。

为了演示如何在混合模型的数值部分中考虑重力荷载和几何非线性，使用地震工程模拟开放系统（OpenSees）（McKenna 1997；Fenves等人，2004）和OpenFresco（见第3章）软件包，进行了混合模拟，其中通过一致地考虑重力荷载引起的二阶效应来测试门式框架。

### 6.3.2 二阶效应

由于二阶效应是由重力荷载引起的，因此了解这些荷载如何影响结构的响应很重要。一般而言，重力荷载在三个层面上影响结构的响应，如下表6.5所示。

表 6.5 重力荷载对结构响应的影响。   


| Level 层级 | Effect 作用（英文原文） | 作用（中文翻译） |
| :--- | :--- | :--- |
| **Section 截面** | Gravity loads cause axial forces that interact with bending moments and shears at the cross section, commonly known as P-M and P-V interaction. | 重力荷载在截面产生轴力，该轴力与截面的弯矩和剪力相互作用，通常称为**轴力-弯矩（P-M）**和**轴力-剪力（P-V）**相互作用效应。 |
| **Element 构件** | Gravity loads affect the stability of structural elements and critical regions, causing softening and local and/or lateral-torsional buckling of elements, commonly known as the small P-Delta (P-δ) effects. | 重力荷载影响结构构件及关键部位的稳定性，导致构件刚度软化、局部屈曲或弯扭屈曲，通常称为**小P-Δ效应（P-δ，构件级二阶效应）**。 |
| **Structure 结构** | Gravity loads influence the stability of a structure, causing additional overturning moments due to large displacements. These effects are commonly known as the large P-Delta (P-Δ) effects. | 重力荷载影响整体结构的稳定性，由大位移产生附加倾覆力矩，通常称为**大P-Δ效应（P-Δ，结构级二阶效应）**。 |

💡 术语说明
- `P-δ`：构件在自身弯曲变形下，轴力产生的附加弯矩，也叫“小二阶效应”。
- `P-Δ`：结构整体侧移引起的重力附加倾覆弯矩，也叫“大二阶效应”。
- `P-M/P-V interaction`：轴力与弯矩、剪力共同作用下，构件截面承载力降低的耦合效应。


为了在有限元软件（如OpenSees）中建模这些非线性几何效应，需要在变形构型中满足构件平衡方程。此外，由于大位移，构件变形与全局构件端部位移之间的协调关系变为非线性。对于框架构件，只要力-变形关系在没有刚体模式的基本坐标系中定义，就可以将这些坐标变换与构件本身的实际实现分开。因此，可以为同一构件实现不同的几何理论而无需修改构件代码。在OpenSees中，CrdTransf软件类通过将任何框架构件的响应量从全局自由度变换到简支基本构件自由度并返回，提供了这一确切功能。CrdTransf类是一个抽象基类，具有用于线性、p-delta和协转几何变换的具体子类。框架构件坐标变换的这种抽象和封装，便于在OpenSees软件框架中在不同线性和非线性坐标变换之间切换，而不会影响框架构件本身的实现。

### 6.3.3 理论和实现

OpenFresco中的二维实验梁柱构件公式（在本案例研究中使用）基于悬臂基本系统的三个并置自由度（见图6.22或第3章）。

位移、速度、加速度和力的必要变换在具体的EEBeamColumn2d类中实现。由于此实验构件类似于解析梁柱构件，因此使用上述OpenSees CrdTransf类将响应量从全局自由度变换到简支基本构件自由度并返回。然而，由于简支基本系统不太适合实验测试（由于两个转动自由度），响应量进一步从简支变换到悬臂基本系统并返回。这些额外的非线性变换直接在EEBeamColumn2d类中实现，并在第3章中详细描述。

图 6.22 实验梁柱构件（EEBeamColumn2d）。

在OpenSees中可用的三种坐标变换（线性、p-delta和协转）中，这里只解释大位移、中等变形的协转变换（de Souza 2000；Filippou和Fenves 2004）。

![](images/6_23.jpg)  
图 6.23 从局部到基本系统A的协转变换。

第一步（未显示），通过应用旋转变换矩阵将梁柱构件的六个端部位移从全局变换到局部坐标系。之后，使用以下公式将局部坐标系中的构件试算位移 $u _ { l , }$ 变换到基本坐标系中的构件变形 $u _ { b }$：

$$
u _ {b, 1} ^ {(A)} = L _ {n} - L \quad L _ {n} = \sqrt {\left(L + u _ {l , 4} - u _ {l , 1}\right) ^ {2} + \left(u _ {l , 5} - u _ {l , 2}\right) ^ {2}}
$$

$$
u _ {b, 2} ^ {(A)} = u _ {l, 3} - \alpha \quad \text {w i t h} \quad \left( \begin{array}{c} u _ {l, 5} - u _ {l, 2} \end{array} \right) \tag {6.1}
$$

$$
u _ {b, 3} ^ {(A)} = u _ {l, 6} - \alpha  = \arctan \left(\frac {u _ {l, 5} - u _ {l, 2}}{L + u _ {l , 4} - u _ {l , 1}}\right)
$$

其中 $L _ { n }$ 是构件弦在变形构型中的长度，$\alpha$ 是弦从构件未变形到变形构型的旋转角。如前面第4章所示，为方便起见在此重复，抗力和构件刚度矩阵从基本系统A到局部坐标系的变换采用以下形式。

$$
\mathbf {p} _ {1} = \mathbf {a} ^ {T} \mathbf {q} ^ {(A)}
$$

$$
\mathbf {k} _ {1} = \frac {\partial \mathbf {p} _ {1}}{\partial \mathbf {u} _ {1}} = \frac {\partial}{\partial \mathbf {u} _ {1}} \left(\mathbf {a} ^ {T} \mathbf {q} ^ {(A)}\right) = \mathbf {a} ^ {T} \mathbf {k} _ {\mathbf {b}} ^ {(A)} \mathbf {a} + \frac {\partial \mathbf {a} ^ {T}}{\partial \mathbf {u} _ {1}} \mathbf {q} ^ {(A)} \tag {6.2}
$$

其中相容性矩阵 $\mathbf{a}$ 和基本坐标系A中的弹性构件刚度矩阵 $\mathbf { k _ { b } } ^ { ( \mathrm { A } ) }$ 由公式(6.3)给出。

$$
\mathbf {a} = \left[ \begin{array}{l l l l l l} - \cos (\alpha) & - \sin (\alpha) & 0 & \cos (\alpha) & \sin (\alpha) & 0 \\ - \frac {\sin (\alpha)}{L _ {n}} & \frac {\cos (\alpha)}{L _ {n}} & 1 & \frac {\sin (\alpha)}{L _ {n}} & - \frac {\cos (\alpha)}{L _ {n}} & 0 \\ - \frac {\sin (\alpha)}{L _ {n}} & \frac {\cos (\alpha)}{L _ {n}} & 0 & \frac {\sin (\alpha)}{L _ {n}} & - \frac {\cos (\alpha)}{L _ {n}} & 1 \end{array} \right] \quad\quad\text{(6.3)}
% \tag {6.3}
$$

$$
\mathbf {k} _ {\mathbf {b}} ^ {(A)} = \left[ \begin{array}{c c c} \frac {E A}{L} & 0 & 0 \\ 0 & \frac {4 E I}{L} & \frac {2 E I}{L} \\ 0 & \frac {2 E I}{L} & \frac {4 E I}{L} \end{array} \right]
$$

$\mathbf { u } _ { \mathbf { l } }$ 和 $\mathbf { p } _ { \mathbf { l } }$ 是局部坐标系中的构件位移和力，${ \mathbf { u _ { b } } } ^ { ( \mathrm { A } ) }$ 和 $\mathbf { q } ^ { ( \mathrm { A } ) }$ 是基本坐标系A中的构件变形和力，$\alpha$ 是弦旋转角，$L _ { n }$ 是构件弦在变形构型中的长度。此外，(6.2)中局部刚度矩阵的第一部分代表材料刚度，第二部分代表几何刚度。

### 6.3.6 测试结果和解释

为了研究重力荷载对门式框架响应的影响，进行了两对模拟。一个模拟在没有施加任何重力荷载的情况下执行，然后在第二个测试中将重力荷载添加到混合模型的解析部分。两个测试都使用了协转几何坐标变换，并且每个测试中U形夹中代表塑性铰的新试样都是相同的。

本节描述了各种结果。首先，讨论在OpenSees中记录的响应。其次，呈现一些在xPC Target机器上记录的预测-校正结果。然后使用STS控制器软件记录的测量来评估测试期间伺服液压控制系统实现的性能。最后，将混合模拟与门式框架模型的纯解析模拟进行比较。

**有限元模型结果：**

层间位移时间历程以及层剪力与层间位移滞回环的比较如图6.26所示。即使没有重力荷载的第一个测试，框架位移和残余位移也很大，但混合模型没有倒塌。在第二次混合模拟中，从组合混合模型表现出的负屈服后切线刚度可以清楚地看出重力荷载的存在。

## 6.4 总结

本章讨论的混合模拟两种新应用，成功展示和验证了第3章提出的新实现的对象导向混合模拟框架的一些功能和整体灵活性。此外，第4章和第5章的几个其他独立开发和实现，如积分方法和事件驱动策略，已被成功用于执行两个具有挑战性的混合模拟案例研究。

第一个应用展示了首次以软实时执行的连续地理分布混合模拟。网络技术的最新进步（Abilene网络）与实验软件框架OpenFresco的结合，使这种实时地理分布混合模拟成为可能。此外，可以得出结论，改进的预测-校正算法和事件驱动策略成功保证了测试的连续执行，以及在网络延迟情况下实验的平滑减速和加速。

讨论了在过去几年中，实时混合模拟已经获得了相当大的重要性，因为许多新型高性能结构装置，如基础隔震支座、耗能装置和高性能材料，表现出固有的速率依赖行为。因此，得出结论，即使混合模拟是地理分布的，对包含此类装置的结构系统进行测试时以正确的加载速率进行也是至关重要的。成功证明，重新设计、扩展和新实现的实验测试软件框架提供了执行此类混合模拟的手段。

第二个案例研究介绍了一种利用混合模拟进行结构倒塌研究的新方法。证明OpenFresco软件框架提供了一套有效的模块，特别是新型实验梁柱构件，用于执行准确、安全且经济有效的结构倒塌混合模拟。此外，证明在混合模型中正确考虑二阶效应的能力对于模拟重力荷载下结构的倒塌至关重要。总之，与常规测试方法相比，结构倒塌行为的混合模拟提供了三个显著优势：（1）重力荷载和由此产生的几何非线性在混合模型的解析部分中用计算机表示，消除了对复杂主动或被动重力荷载设置的需求；（2）不需要保护昂贵的测试设备免受倒塌期间试样冲击的影响，因为执行器控制系统将限制测试试样的移动；（3）只有结构的关键的、对倒塌敏感的元素需要物理测试，允许在相同预算下大幅增加不同倒塌测试的数量。最后，再次详细讨论了门式框架结构、测试设置和程序、传动系统性能和混合模型的响应。
